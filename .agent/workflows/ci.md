# CI/CD Reasoning Workflow
Version: 1.1.0

Command: /ci

## Objective
Understand, evaluate, and improve CI/CD logic without
forcing automation or breaking existing setups.

---

## Step 1: Repository Scan
Detect:
- Languages
- Test frameworks
- Build tools
- Existing CI config

---

## Step 2: CI Understanding
Summarize:
- What CI exists
- What assumptions are implicit
- What is missing (if anything)

---

## Step 3: Effectiveness Evaluation
Evaluate:
- Flakiness
- Overhead
- Developer friction

---

## Step 4: Improvement Proposal (Optional)
If requested:
- Propose minimal improvements
- No changes without approval

---

## Step 5: Implementation (Optional)
If proposal approved:
- Apply changes to configuration or code
- Verify build impact

---

## Output Format
1. CI Summary
2. Gaps & Risks
3. Optional Improvements
4. Implementation Results (if applicable)
